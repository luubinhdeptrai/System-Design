Write production-quality, clean, and readable code.

Requirements:

* Use clear, descriptive variable and function names (self-explanatory).
* Follow single responsibility principle (each function does one thing).
* Avoid complex one-liners; prefer explicit and readable logic.
* Structure the code into small, well-organized functions/modules.
* Add concise comments explaining *why*, not obvious *what*.
* Avoid magic numbers; use named constants.
* Handle edge cases and errors clearly.
* Keep consistent formatting and style.
* Make the code easy to understand for another developer reading it for the first time.

After writing the code:

* Briefly explain the structure and design decisions.
* Highlight any trade-offs or assumptions.

Refactor the code once before finalizing to improve readability and maintainability.
